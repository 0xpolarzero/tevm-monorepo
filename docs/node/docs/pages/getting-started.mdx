# Getting Started

Welcome to Tevm Node! This guide will help you get started with running Ethereum in JavaScript.

## Installation

```bash
npm install tevm
```

## Quickstart

To get started quick just use the viem api

```typescript
// createMemoryClient creates a viem client using in memory TevmNode as backend
import {createMemoryClient} from 'tevm'

// create a viem client using tevmNode as backend
// note: tree shakable versions are available too
const client = createMemoryClient()

// example of using a viem api
console.log(await client.getBlockNumber())

// The tevm node exists on client.transport and handles json rpc requests to the transport
const internalTevmNode = client.transport.tevm
```

## Usage Examples

### Basic Usage with Tevm Node

The lowest level API for Tevm is called [TevmNode](https://github.com/evmts/tevm-monorepo/blob/main/packages/node/src/TevmNode.ts). It is an ethereum node running in JavaScript.

You have access to the low level primitives such as [await node.getVm()](./reference/vm.mdx) but most of the time you want to use `actions`.

Tevm has a concept of `actions` (similar to viem) that can be imported from [`tevm/actions`](https://github.com/evmts/tevm-monorepo/blob/main/packages/actions/docs/globals.md#functions) package for use with `TevmNode`. There are high level actions such as `tevmCall`. There are also actions that implement the JSON-RPC API.

Note: This is the lowest level API and higher level APIs exist for usage with Viem and Ethers below. But for both those apis they still have escape hatches for accessing the TevmNode directly for advanced functionality. For a viem client you can access tevm node via `client.transport.tevm`

TevmNode without using viem or ethers can be desired if building libraries or trying to use the most minimal api.

```typescript
import { createTevmNode } from 'tevm'
import { callHandler, setAccountHandler, getAccountHandler } from 'tevm/actions'

const node = createTevmNode()

// Execute a contract call
const result = await callHandler(node)({
  to: '0x1234567890123456789012345678901234567890',
  data: '0x...',
  value: 0n
})

// Set account state
await setAccountHandler(node)({
  address: '0x1234567890123456789012345678901234567890',
  balance: 1000000000000000000n // 1 ETH
})

// Get account state
const account = await getAccountHandler(node)({
  address: '0x1234567890123456789012345678901234567890'
})
```

### Using with Viem

Tevm itself is built on top of viem using viem patterns such as the `actions` pattern. This makes the viem api the most recomended API.

IMPORTANT: Most of these docs do not use the viem api because the viem api is documented in [viem](https://viem.sh). These docs focus on the new apis introduced by Tevm.

See [Tevm viem guide](./examples/viem.mdx) for more.

```typescript
import { createMemoryClient, http } from 'tevm'
import { optimism } from 'tevm/common'

const client = createMemoryClient({
  fork: {
    transport: http('https://mainnet.optimism.io'),
    blockTag: 'latest',
  },
  common: optimism,
})

// Use viem's public actions
const block = await client.getBlock({ blockTag: 'latest' })

// Mix with Tevm actions
await client.tevmSetAccount({
  address: '0x1234567890123456789012345678901234567890',
  balance: 1000000000000000000n
})
```

### Using with Ethers

```typescript
import { createTevmNode } from 'tevm'
import { requestEip1193 } from 'tevm/decorators'
import { BrowserProvider } from 'ethers'

const node = createTevmNode().extend(requestEip1193())
const provider = new BrowserProvider(node)

const block = await provider.getBlockNumber()
const balance = await provider.getBalance('0x1234...')
```

## Package Documentation

Tevm is modular - install the full package with `npm install tevm` or individual packages like `@tevm/actions`. Import from specific packages:

```typescript
// Full package
import { createTevmNode } from 'tevm'

// Individual packages
import { createTevmNode } from '@tevm/node'
import { callHandler } from '@tevm/actions'
```

See our [Package Reference](./api/packages) for detailed documentation of each module.
